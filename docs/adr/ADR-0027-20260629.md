# ADR-0027: Implement Circuit Breaker Pattern for External API Calls

**Status:** Accepted
**Date:** 2026-06-29
**Author:** P-r-e-m-i-u-m
**Reviewers:** Engineering Team

---

## Context

Our service integrates with 4 external APIs. When any of these APIs degraded, our service would also degrade as requests piled up waiting for timeouts. A 2-minute payment API outage caused a 15-minute degradation of our entire checkout flow.

## Decision

We will implement the Circuit Breaker pattern for all external API calls. The breaker opens after 5 consecutive failures and attempts recovery after 60 seconds in half-open state.

## Alternatives Considered

| Option | Pros | Cons | Decision |
|-|-|-|-|
| Chosen approach | Best fit for requirements | Some tradeoffs | Accepted |
| Alternative A | Simpler | Missing key features | Rejected |
| Alternative B | More features | Too complex / costly | Rejected |

## Consequences

### Positive
External API failures no longer cascade to our service. Checkout flow remains available during payment API degradation.

### Negative
Circuit breaker state must be monitored and alerted on OPEN state.
