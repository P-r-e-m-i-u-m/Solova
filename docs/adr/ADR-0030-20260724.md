# ADR-0030: Use Structured JSON Logging Over Plain Text Logs

**Status:** Accepted
**Date:** 2026-07-24
**Author:** P-r-e-m-i-u-m
**Reviewers:** Engineering Team

---

## Context

Debugging production issues was taking 3-4x longer than necessary because log entries were unstructured plain text. Correlating logs across services required manual grep patterns and was error-prone.

## Decision

We will adopt structured JSON logging with a custom Logger class supporting log levels, context inheritance, child loggers for request tracing, and file/console transports.

## Alternatives Considered

| Option | Pros | Cons | Decision |
|-|-|-|-|
| Chosen approach | Best fit for requirements | Some tradeoffs | Accepted |
| Alternative A | Simpler | Missing key features | Rejected |
| Alternative B | More features | Too complex / costly | Rejected |

## Consequences

### Positive
Production debugging time reduced by 70%. Log aggregation pipeline now fully automated. Request tracing across services via X-Request-ID correlation.

### Negative
Log files require rotation policy to prevent disk exhaustion.
