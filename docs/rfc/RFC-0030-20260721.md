## Summary

This RFC proposes a multi-region deployment strategy across 3 geographic regions to reduce latency for international users and provide disaster recovery with automatic failover.

## Motivation

All users in Asia/Pacific experience 180-250ms baseline latency. Single region is a single point of failure. No DR capability. EU compliance requires data residency.

## Detailed Design

Target regions: ap-south-1 (primary), us-east-1 (secondary), eu-west-1 (tertiary). PostgreSQL: primary with read replicas. Redis: independent cluster per region. RTO: 5 minutes, RPO: 30 seconds.

## Alternatives Considered

CDN only: reduces static latency but not API latency. Single region better hardware: vertical scaling has limits. Cloudflare Workers: limited to stateless workloads.
