## Executive Summary

Queue processor memory usage grew unbounded causing OOM restart. 1,847 jobs were delayed. No data loss due to Redis persistence.

## Root Cause Analysis

setTimeout callbacks in retry logic closed over full job objects. Under sustained load with many retries, thousands of job objects accumulated in heap memory. The leak required 6+ hours of sustained load to become critical.

## Resolution

Restarted queue processor to release accumulated memory. Developed fix extracting setTimeout callback to named function with timer.unref(). Deployed and monitored over 48h — memory growth under 1MB/hour.
