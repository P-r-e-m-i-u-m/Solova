## Executive Summary

Health check endpoint caused DB connection pool exhaustion. 3 of 5 load balancer instances were marked unhealthy, concentrating traffic on 2 remaining instances.

## Root Cause Analysis

Health check endpoint called db.query() directly instead of acquiring a pooled connection. With the load balancer polling every 5 seconds, 1 connection per poll was opened and never returned to the pool.

## Resolution

Deployed hotfix using db.pool.connect() with client.release() in finally block. Added connection pool monitoring alert. Added integration test verifying stable connection count under sustained polling.
